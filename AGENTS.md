# Gomi's Golden Days 작업 규칙

## 1. 작업 경계

- 모든 작업은 이 저장소 안에서만 한다.
- 시작할 때 `git status`, 현재 브랜치, 최신 `origin/main`을 확인한다.
- 관련 없는 사용자 변경과 미추적 폴더는 그대로 보존한다.
- 모호한 요청은 가정하지 말고, 해석과 선택지를 짧게 제시한 뒤 판단을 기다린다.
- 다단계 작업은 성공 조건과 검증 기준을 먼저 밝힌다.
- 요청한 기능만 최소 수정하며, 범위 밖 디자인 변경·리팩터링·포맷 정리는 하지 않는다.
- Steve가 명시하지 않으면 기존 랜딩페이지와 정상 동작하는 디자인을 유지한다.
- 새 소스 파일의 첫 줄에는 역할을 설명하는 한국어 주석 한 줄을 넣는다.
- 오류는 실제 로그와 스택트레이스를 확인한 뒤 대응하며 추측으로 수정하지 않는다.
- 코드를 바꾸면 테스트한다. 테스트가 없으면 빌드 또는 실행을 확인한다.
- UI 변경은 데스크톱과 모바일에서 검증한다.

## 2. 승인과 게시

- 게시 전 로컬 미리보기 또는 diff와 검증 결과를 먼저 보고한다.
- Steve가 **“ㅇㅋ 커밋”**이라고 명시하기 전에는 커밋하지 않는다.
- 커밋 뒤 푸시는 자동 진행하지 않으며 별도 승인을 받는다.
- 승인받지 않은 푸시, PR 생성·수정, 병합, 배포는 하지 않는다.
- Steve가 **“ㅇㅋ 병합”**이라고 명시하기 전에는 병합하거나 배포하지 않는다. **“배포ㄱㄱ”**만으로는 병합 승인으로 보지 않는다.
- 승인받은 파일만 명시적으로 스테이징하고 staged diff를 확인한다.
- 병합 후 실제 공개 사이트와 핵심 경로를 확인해 결과를 보고한다.
- 인증이나 권한으로 막히면 Steve가 해야 할 정확한 한 가지 행동만 안내한다.
- 안전한 PowerShell, 경로 처리, 인증 확인, 반복 GitHub 작업은 Codex가 직접 수행한다.

## 3. 판단과 소통

- 쉬운 한국어로 `결론 → 이유 → 다음 행동` 순서로 짧게 말한다.
- 트래픽 증가, 매출 증가, 리스크 감소, Steve의 시간 절약 순으로 실제 효과를 판단한다.
- 일반론보다 현재 병목을 먼저 다루고, 이미 답이 있는 질문은 반복하지 않는다.
- 저장소와 공개 사이트에서 안전하게 확인·처리할 수 있는 일은 먼저 수행한다.
- Steve의 로그인, 결제, 계정 권한, 민감한 판단만 별도 체크리스트로 남긴다.
- 큰 작업의 보류·삭제·순서 변경, 새 도구 도입은 근거와 선택지를 제시하고 Steve가 결정하게 한다.

## 4. 사이트 기술 규칙

- GitHub Pages 호환성과 상대 경로 에셋 참조를 유지한다.
- 홈페이지와 `/dog-age-calculator/`가 계산기 로직을 공유하므로 관련 변경 시 두 페이지를 함께 검증한다.
- 공유 내비게이션·푸터 변경은 홈페이지와 하위 페이지에서 일관성과 모바일 가로 넘침을 확인한다.
- SEO 상태는 `robots.txt`, `sitemap.xml`, canonical, 내부 링크, 공개 HTTP 응답을 먼저 확인한다.
- Search Console에서 같은 URL의 색인 요청을 반복하지 않는다. 여러 페이지는 sitemap 재제출을 우선한다.
- Search Console 클릭은 Steve가 명시적으로 승인하고 인증된 브라우저가 연결된 경우에만 대신 수행한다.

## 5. 채널 운영 금지선

- Cloudflare Web Analytics가 이미 사용 중임을 전제로 한다.
- GA4·UTM 같은 추가 계측을 자동으로 P0에 올리지 않는다. 정밀한 채널 귀속이 실제 의사결정에 필요할 때만 제안한다.
- Instagram 외부 링크의 역할은 별도 지시 전까지 유지한다.
  1. Dog Age Calculator → 계산기 전용 페이지
  2. Golden Days Journal → 메인 랜딩페이지 → Gumroad 또는 Etsy 구매
  3. YouTube → Gomi YouTube 채널
- 저널 링크를 마켓플레이스 직행으로 임의 변경하지 않는다.
- Pinterest는 신규 핀을 성과 확인 전에 삭제하거나 반복 재게시하지 않는다.
- Instagram 자동 게시를 유지하고 Steve에게 Pinterest 수동 반복 작업을 요구하지 않는다.
- Pinterest 진단은 검색 결과가 아니라 Analytics와 Search Console의 노출·저장·클릭 데이터로 판단한다.

## 6. 현재 운영 인계 — 2026-07-21 기준

- 저장소: `gratefulenlightenment/gomisgoldendays`
- 공개 사이트: `https://gomisgoldendays.com/`
- Cloudflare Pages 배포와 공개 핵심 경로, 랜딩 영상, 계산기 2곳, `#calculator`, SEO 기본 파일은 정상 검증됨.
- Search Console: sitemap에 12개 페이지가 발견됨. 홈페이지는 색인됨. Golden Walk는 색인 요청 1회 완료했으므로 재요청 없이 상태만 추후 확인한다.
- Pinterest: 도메인 Claim, 공개 설정, Instagram 자동 게시, 보드 설명 SEO 개선 완료. 예약 핀 게시 여부와 48시간·7일 노출 추이를 확인하고 원본 2:3 핀 재고를 늘린다.
- YouTube: 계산기 직행 링크, 메인 사이트·Instagram·Etsy 링크, 지속형 채널 설명 반영 완료.
- Etsy·Gumroad: 상품 공개·구매, 지속형 소개, 핵심 상품 정보와 신뢰 요소 반영 완료.
- Beehiiv: 첫 Web-only 글, Archive, 구독 폼, Welcome Email, Tiny Changes Checklist 전달이 정상. 기본 기능으로 충분하므로 유료 Automation은 도입하지 않는다.
- Instagram: 지속형 Bio와 계산기·메인 랜딩·YouTube 링크 3개가 정상이다.
